🔑 Why some PST files cannot be opened in a browser

PST files use one of two encryption methods. The browser handles one but not the other:

✓ Compressible Encryption (NDB)
Uses a fixed substitution table published in Microsoft's open [MS-PST] specification. Fully reversible in the browser. Common in older Outlook and personal archive PSTs.
✕ High Encryption (AES/DES)
Uses a key derived from your Windows user profile — a key that physically never leaves your Windows installation. Cannot be decrypted anywhere else, including any browser or WASM runtime, without that exact Windows machine.

Most enterprise eDiscovery exports (Microsoft 365, Exchange Server) use High Encryption. For those, the manual tools below work because they run on the same machine holding the key.
